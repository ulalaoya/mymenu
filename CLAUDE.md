# MyMenu — הנחיות עבודה ל-Claude Code

פרויקט: אפליקציית PWA בעברית (RTL) לתכנון ורישום תפריט יומי לילדה בת 11.
**האפיון המלא והמחייב נמצא ב-`SPEC.md` — קרא אותו במלואו לפני כל עבודה.**

---

## חלוקת עבודה בין מודלים (חובה!)

| שלב | מודל | איך |
|---|---|---|
| תכנון, ארכיטקטורה, פירוק למשימות | **Fable 5** (`claude-fable-5`) | הסשן הראשי / Plan mode רץ על Fable 5 |
| **כתיבת הקוד — כל המימוש** | **Opus 4.8** (`claude-opus-4-8`) | כל משימת מימוש נשלחת ל-subagent עם `model: opus` |
| בדיקות, QA, סקירת UX, אימות מול האפיון | **Fable 5** | הסשן הראשי מריץ את הבדיקות ובוחן את התוצאה |

כללים:
- Fable 5 **לא כותב קוד מוצר** — הוא מתכנן, מנסח משימות מדויקות ל-Opus, ואז מאמת.
- כל משימה ל-Opus 4.8 חייבת לכלול: קבצים רלוונטיים, הקטע הרלוונטי מ-SPEC.md, וקריטריון סיום בדיק.
- אחרי כל שלב: Fable 5 מריץ את הטסטים + בדיקה ידנית ב-preview (viewport מובייל 390×844) ומוודא עמידה בקריטריוני הקבלה שבסוף SPEC.md לפני מעבר לשלב הבא.
- אם Opus סטה מהאפיון או מהעיצוב — מחזירים לו משימת תיקון, לא מתקנים ידנית.

## שלבי ביצוע (Milestones)

1. **שלד**: Vite + React + TS + PWA + RTL + מערכת עיצוב (סעיף 9 ב-SPEC) + חיתוך האייקונים מה-PNG.
2. **פרופילים וכניסה**: הרשמה, hash סיסמאות, זיהוי מכשיר, ריבוי פרופילים מבודדים (סעיף 3).
3. **מאגר מזון + מודל נתונים**: seed של 150+ מאכלים בעברית, סכמת Dexie (סעיפים 4, 7).
4. **מנוע ההמלצות**: אלגוריתם הניקוד, אילוצי תזונה וגיוון, למידת שעות, cold-start (סעיף 5) — **עם unit tests באותה משימה**.
5. **מסכים**: בית, תפריט יומי, רישום ארוחה, דירוגים, מים, טיפ יומי (סעיף 6).
6. **היסטוריה והתקדמות + הגדרות**.
7. **QA סופי**: Playwright E2E, Lighthouse, בדיקת RTL, מעבר על כל קריטריוני הקבלה.

## עקרונות שאסור להפר

- **עברית בלבד** בממשק. `dir="rtl"` בכל מקום. אפס מחרוזות אנגלית למשתמשת.
- **חוויה חיובית בלבד**: אין קלוריות, אין משקל, אין דיאטה, אין ביקורת. רק עידוד. הממתק היומי הוא פיצ'ר, לא פשרה.
- **שובע = המדד החשוב ביותר** במנוע ההמלצות (משקל 3.0 בניקוד).
- **גיוון**: מנה עיקרית לא חוזרת בטווח 3 ימים.
- **הכל מקומי**: אין קריאות רשת, אין API, אין analytics. IndexedDB בלבד.
- **Look & feel** אך ורק לפי הפלטה והסגנון ב-SPEC.md סעיף 9 ולפי קבצי ה-PNG בתיקייה.
- קוד ב-TypeScript מלא, לוגיקת המנוע מופרדת מה-UI (טסטבילית).

## פקודות

```bash
npm run dev        # שרת פיתוח
npm run test       # Vitest
npm run test:e2e   # Playwright
npm run build      # בניית production + PWA
```
