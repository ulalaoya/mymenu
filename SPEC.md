# MyMenu — אפיון מלא (Product Spec)

> אפליקציית PWA לתכנון ורישום תפריט אוכל יומי לילדה בת 11.
> שפה: עברית בלבד, RTL. נתונים: מקומיים בלבד על המכשיר. ללא שרת, ללא עלויות.

---

## 1. מטרת המוצר

אפליקציה אישית וחיובית שעוזרת לילדה בת 11:
1. **לקבל המלצה לתפריט יומי מלא** — מותאם להעדפות שלה ומאוזן תזונתית לפי מה שילדים בגילה צריכים.
2. **לרשום מה אכלה בפועל ומתי** — בקלות ובכיף, בלי תחושת מטלה.
3. **לדרג** את הארוחות (טעם + שובע) כדי שהאפליקציה תלמד ותשתפר.
4. **לקבל טיפ יומי** לשמירה על אורח חיים בריא.

### עקרון-על: חוויה חיובית בלבד
- **אין** ספירת קלוריות, **אין** שקילה, **אין** מסרים על משקל או דיאטה, **אין** "אכלת יותר מדי".
- השפה תמיד מעודדת: "כל הכבוד!", "איזה יופי של גיוון!", "נסי מחר גם ירק ירוק 🥦".
- ממתק אחד ביום הוא חלק לגיטימי ומתוכנן מהתפריט — לא "חטא".
- המטרה: יחס בריא ושמח לאוכל, תחושת מסוגלות ובעלות של הילדה על הבחירות שלה.

## 3. משתמשים וכניסה

### 3.1 פרופילים מרובים
- האפליקציה תומכת בכמה ילדים על אותו טלפון. כל פרופיל מבודד לחלוטין (נתונים, העדפות, היסטוריה).
- פרופיל = שם משתמש + סיסמה + אווטאר (בחירה מתוך סט חמוד) + צבע אישי.

### 3.2 התחברות וזיהוי מכשיר
- הרשמה: שם משתמש, סיסמה (מינימום 4 תווים — זו ילדה, לא בנק), שאלת שחזור פשוטה ("מה שם חיית המחמד שלך?").
- הסיסמה נשמרת כ-hash (SHA-256 + salt) ב-IndexedDB. לעולם לא בטקסט גלוי.
- **זיהוי מכשיר**: אחרי התחברות מוצלחת נשמר token מקומי — בפתיחה הבאה מאותו טלפון נכנסים ישר לפרופיל האחרון, בלי סיסמה.
- מסך "החלפת משתמש" מציג את כל הפרופילים כקלפים עם אווטאר; מעבר לפרופיל אחר דורש את הסיסמה שלו.
- שחזור סיסמה: דרך שאלת השחזור.

## 4. מאגר המזון (Seed Data)

מאגר מובנה של **150+ מאכלים ישראליים** מוכרים לילדים, בעברית. כל פריט:

```ts
interface FoodItem {
  id: string;
  name: string;            // "פסטה ברוטב עגבניות", "שניצל", "קוטג'", "מלפפון"
  emoji: string;           // 🍝 🍗 🥒
  category: MealSlot[];    // לאילו ארוחות מתאים: בוקר / עשר / צהריים / מנחה / ערב / ממתק
  foodGroups: FoodGroup[]; // ראה בהמשך
  tags: string[];          // "חם", "קר", "כריך", "מבושל", "מהיר"
}

type FoodGroup =
  | "פחמימות"      // לחם, פסטה, אורז, תפו"א, קוסקוס
  | "חלבונים"      // עוף, בשר, דגים, ביצים, טונה
  | "מוצרי חלב"    // גבינות, יוגורט, חלב
  | "קטניות"       // חומוס, עדשים, אפונה, שעועית
  | "ירקות"
  | "פירות"
  | "שומנים בריאים" // אבוקדו, טחינה, אגוזים, שמן זית
  | "מתוקים";       // הממתק היומי
```

### יעדים תזונתיים יומיים (גיל 9–13, מוצגים כ"מטרות היום" ידידותיות)
- 5 קבוצות מזון לפחות ביום, מתוכן: ירק אחד לפחות ×2, פרי ×2, חלבון בכל ארוחה עיקרית.
- **שתייה: 6–8 כוסות מים** (~1.5–2 ליטר) — מד מים ויזואלי שמתמלא.
- ממתק משמעותי אחד ביום — הילדה בוחרת איזה ומתי.
- הצגה כצ'קליסט צבעוני של קבוצות מזון, לא כמספרים.

## 5. מנוע ההמלצות (הלב של האפליקציה)

### 5.1 מבנה תפריט יומי מומלץ
| ארוחה | שעה מומלצת (ברירת מחדל) |
|---|---|
| ארוחת בוקר | 07:00 |
| הפסקת עשר | 10:00 |
| ארוחת צהריים | 13:30 |
| ארוחת מנחה (+הממתק היומי) | 16:00 |
| ארוחת ערב | 19:00 |

- השעות הן ברירת מחדל; המנוע **לומד** את שעות האכילה האמיתיות מהרישומים ומעדכן את ההמלצות (ממוצע נע של 14 יום אחרונים).
- בין ארוחות יש הפסקות — האפליקציה לא מציעה אוכל בין החלונות.

### 5.2 אלגוריתם הניקוד (בחירת מאכל לכל משבצת)
לכל מאכל מחושב ציון:

```
score = 3.0 × satietyAvg        // דירוג שובע ממוצע (1–5) — המשקל הגבוה ביותר!
      + 2.0 × tasteAvg          // דירוג טעם ממוצע (1–5)
      + 1.5 × actuallyAteRate   // כמה פעמים נאכל בפועל מתוך הפעמים שהוצע
      + 1.0 × nutritionFit      // כמה המאכל משלים קבוצות מזון שחסרות היום
      - 2.5 × recencyPenalty    // הוצע/נאכל ב-3 הימים האחרונים → קנס גיוון
      - 4.0 × dislikedFlag      // סומן "לא אהבתי" (דירוג ≤ 2) → כמעט לא יוצע
```

- **שובע הוא המלך**: תפריט עם ציון שובע גבוה מקבל עדיפות ברורה בהצעות עתידיות.
- **גיוון**: מנה עיקרית לא חוזרת בטווח של 3 ימים; לפחות מאכל "חדש/נדיר" אחד ביום כדי להרחיב טעמים.
- **אילוץ תזונתי**: התפריט היומי חייב לכסות את כל קבוצות המזון מסעיף 4.
- למשתמשת חדשה (cold start): שאלון היכרות קצר וכיפי — "מה את אוהבת?" עם קלפי מאכלים לסימון לב ❤️ / איקס — שמזין את הציונים ההתחלתיים.

### 5.3 לולאת הלמידה
1. בבוקר: האפליקציה מציגה תפריט מומלץ ליום.
2. הילדה יכולה **להחליף מאכל** בכל משבצת (מוצגות 3 חלופות מדורגות) — ההחלפה עצמה היא סיגנל העדפה.
3. אחרי כל ארוחה: רישום מה נאכל בפועל (מההמלצה / משהו אחר / דילגה) + שעה.
4. דירוג: טעם ⭐1–5, **שובע** (סקאלת 5 פרצופים — מ"עדיין רעבה" עד "שבעה ומרוצה"), ואופציונלי מצב רוח אחרי הארוחה.
5. מאכלים שהילדה רשמה ידנית (חדשים) נוספים אוטומטית **למאגר האישי שלה** וזמינים להמלצות עתידיות.
6. תפריט יומי שקיבל ציון כולל גבוה נשמר כ"תפריט מנצח" ⭐ — המנוע ישתמש בו כתבנית בעתיד (עם וריאציות לשמירת גיוון).

### 5.4 טיפ יומי
- מאגר מובנה של **60+ טיפים** קצרים, חיוביים ומותאמי גיל: שתייה, ארוחת בוקר, תנועה, שינה, ירקות בצבעים, אכילה איטית ועוד.
- טיפ אחד ביום, מתחלף בחצות, לא חוזר עד שנגמר הסבב. מוצג בכרטיס בולט במסך הבית עם אייקון הנורה/נצנוץ.

## 6. מסכים (User Flows)

### 6.1 בחירת פרופיל / התחברות
קלפי פרופילים עם אווטארים → כניסה אוטומטית לפרופיל האחרון → כפתור "פרופיל חדש".

### 6.2 מסך הבית (Home) 🏠
- ברכה אישית לפי שעה: "בוקר טוב, נועה! ☀️"
- **התפריט של היום** — ציר זמן אנכי של 5 הארוחות + שעות. כל ארוחה: אייקון, שם, סטטוס (עתידית / הגיע הזמן! / נאכלה ✓ / דולגה).
- מד מים אינטראקטיבי — כוסות שמתמלאות בלחיצה 💧.
- כרטיס הטיפ היומי.
- צ'קליסט קבוצות המזון של היום (מתמלא בצבע ככל שנרשמות ארוחות).
- כפתור צף ➕ "רשמי מה אכלת".

### 6.3 מסך התפריט היומי 📖
- פירוט כל ארוחה: המאכלים, קבוצות המזון (תגיות צבעוניות), שעה מומלצת.
- החלפת מאכל: לחיצה → bottom sheet עם 3 חלופות + חיפוש חופשי במאגר.
- בחירת הממתק היומי 🧁 מתוך רשימת המתוקים.
- כפתור "ערבבי לי מחדש" 🔄 — מגריל תפריט חלופי (בכפוף לאילוצים).

### 6.4 רישום ארוחה ✏️
- "מה אכלת?" — בחירה מהתפריט המומלץ בלחיצה אחת, או חיפוש/הוספה חופשית (מאכל חדש נשמר למאגר האישי).
- "מתי?" — ברירת מחדל עכשיו, אפשר לשנות.
- דירוג טעם ⭐⭐⭐⭐⭐ + דירוג שובע (5 פרצופים) + מצב רוח (אופציונלי).
- אנימציית חגיגה קטנה (קונפטי עדין) אחרי רישום + משפט מעודד.

### 6.5 היסטוריה והתקדמות 📊
- לוח שנה חודשי — כל יום נצבע לפי שלמות הרישום.
- לחיצה על יום → מה נאכל, דירוגים, כמה מים.
- מסך "הקירות שלי": רצף ימים (streak) 🔥, מאכלים חדשים שניסיתי, התפריטים המנצחים שלי ⭐.
- גרף פשוט וידידותי של גיוון קבוצות מזון שבועי (לא קלוריות!).

### 6.6 הגדרות ⚙️
- עריכת פרופיל (שם, אווטאר, צבע).
- שעות ארוחות מועדפות (דריסה ידנית של הלמידה).
- אלרגיות / מאכלים אסורים (סינון קשיח מהמנוע) + צמחונית כן/לא.
- החלפת משתמש / יציאה.

## 9. מערכת עיצוב (Look & Feel)

לפי הלוגו והאייקונים המצורפים:

```css
:root {
  --blue:       #5B9BD5;   /* כחול ידידותי — קווי מתאר, טקסט ראשי */
  --blue-light: #DCEBFA;   /* רקעי כרטיסים כחלחלים */
  --coral:      #F0806C;   /* אלמוגי — הדגשות, לבבות, CTA משני */
  --coral-light:#FBDDD6;
  --yellow:     #F5B041;   /* צהוב-שמש — כוכבים, נצנוצים, הדגשות שמחות */
  --bg:         #FAFBFC;   /* רקע כללי לבן-שבור */
  --card:       #FFFFFF;
  --text:       #3D5A80;   /* כחול כהה לטקסט */
  --success:    #7DCEA0;
}
```

- **אווירה**: רכה, מאירה, אופטימית. הרבה אוויר לבן, כרטיסים לבנים עם `border-radius: 24px` וצל עדין מאוד.
- **מוטיבים חוזרים**: עלה כחול קטן 🍃, נצנוץ צהוב ✦ — בדיוק כמו באייקונים.
- **טיפוגרפיה**: Rubik; כותרות עגלגלות ושמחות, גוף 16px+.
- **כפתורים**: גדולים, מעוגלים מאוד (pill), ידידותיים לאצבעות של ילדה.
- **אנימציות**: עדינות ומתגמלות — מילוי כוס מים, קונפטי ברישום, כוכבים שנדלקים בדירוג.
- כל אזורי המגע ≥ 44px. ניגודיות טקסט AA לפחות.

## 10. דרישות איכות ובדיקות

### בדיקות אוטומטיות (חובה, נכתבות תוך כדי פיתוח)
1. **Unit (Vitest)**: אלגוריתם הניקוד (כולל קנס גיוון ו-boost שובע), אילוצי קבוצות המזון, למידת שעות, hash סיסמאות, סינון אלרגיות.
2. **E2E (Playwright, viewport מובייל)**: הרשמה → תפריט יומי מוצג → החלפת מאכל → רישום ארוחה + דירוג → המאכל המדורג-נמוך לא מופיע מחר; זיהוי מכשיר (רענון לא דורש סיסמה); בידוד בין שני פרופילים; עבודה אופליין.
3. **בדיקת RTL**: אין טקסט חתוך, אין אלמנטים "הפוכים".

### קריטריוני קבלה (Definition of Done)
- [ ] מותקנת כ-PWA ועובדת אופליין מלא אחרי טעינה ראשונה.

- [ ] מנה עיקרית לא חוזרת יומיים ברצף.
- [ ] מאכל חדש שנרשם ידנית מופיע בהצעות עתידיות.
- [ ] אפס טקסטים באנגלית בממשק; אפס אזכורי קלוריות/משקל/דיאטה.
